
Attention(Q, K, V) = softmax(Q*K^T / sqrt(d_k)) * V

X - входная матрица из векторов эмбеддингов [seq_len, d_model]

Матрицы проекций: W_Q, W_K [d_model, d_k], W_V [d_model, d_v]

Q = X * W_Q - Query	[seq_len, d_k]
K = X * W_K - Key	[seq_len, d_k]
V = X * W_V - Value	[seq_len, d_v] (d_k == d_v)

S = Q*K.T		[seq_len, seq_len]	- билинейная форма по X.
P = softmax(S)	[seq_len, seq_len]	- функция активации после билинейного слоя.
O = P*V			[seq_len, d_v]		- результат Attention.





S = Q*K.T/sqrt(d_k) [seq_len, seq_len] - билинейная форма по X. 

Это позволяет:
1. Не привязывать матрицы весов к динамическому размеру seq_len(X) (как например в случае X*W_example)
	Q*K.T = (X*W_Q)*(X*W_K).T = X * W_Q*W_K.T * X
	W_Q*W_K.T [d_model, d_model] - веса не зависят от seq_len
2. Описывать более сложные модели связи в X (так как форма билинейная)
3. Веса НЕ объединяются в одну матрицу: 
	W = W_Q*W_K.T [d_model, d_model]
	Объединение в одну матрицу резко увеличивает количество необходимых к обучению весов.
	W_Q и W_K.T являются проектирующими матрицами с меньшим количеством весов (2*d_model*d_k)

Примечание по sqrt(d_k):
1. Рассмотрим вектора q (Query) и k (Key) как независимые случайные величины E(q,k) = 0, D(q,k) = 1.
	Тогда q*k = sum_h(q_h*k_h). 
	D(q*k) = sum_h(D(q*k)) = d_k
	Тогда стандартное отклонение std(q*k) = sqrt(d_k)
2. Необходимо деление на sqrt(d_k), так как последующий softmax очень чувствителен 
	к относительному разбросу значений.





P = softmax(S)

softmax(S)_ij = exp(S_ij) / (sum_k (exp(S_ik)))

Плюсы softmax:
1. Гладкая, бесконечно дифференцируемая
	Является дифференцируемой аппроксимацией hard argmax (реализованной через exp).
2. Производная softmax может быть выражена через сам softmax (для backpropagation grad)
	dS_i / da_j = S_i * (delta_ij - S_j)
3. Численная стабильность: 
	softmax(T) = softmax(T - c) - обеспечивает отсутствие переполнения при взятии exp(float)
3. Нормирующая выходные вектора, sum(P_i) = 1; P_i in [0,1];
	Таким образом, реализуется: 
	Выпуклая оболочка, построчная нормировка, интерпретация P_i как вероятности.

Таким образом, softmax в attention является функцией активации.





O = P*V - выходная матрица Attention-слоя.

1. Матрица P является результатом функции активации. 
	Или иначе матрицей связей (вероятностей) [seq_len, seq_len].
	Необходимо перевести признаки из (0,1) в R^(seq_len, d_v).
	O = P*V является таким переводом.
2. V = X * W_V - зависимость V(X) необходима:
	Без зависимости V(X) результат Attention всегда был бы комбинацией константных строк W_V с различной вероятностью P.





Плюсы самого Attention:
1. До этого применялись recurrent NN:
	1.1. h_t = f(h_(t-1), x_t)
	1.3. Для генерации 100 токенов требуется 100 последовательных(!) вызовов RNN.
2. Attention совершает матричное умножение, которое можно распараллелить.
	2.1. Так как матричное умножение параллелизированно, то Attention не знает последовательность слов.
	2.2. Для encoder : Attention
	2.3. Для decoder : Masked Attention (softmax(z) = -inf) для Causal Mask
